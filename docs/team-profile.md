## Team Working Agreement

### Meeting Schedule
- Weekly team meeting every Friday after class.
- Additional meetings may be scheduled before major milestones or presentations.

### Communication Channel
- LINE Group for daily communication.
- GitHub Issues and Discussions for project-related tasks and decisions.

### Task Management
- Tasks will be assigned during weekly meetings.
- Each member is responsible for updating progress before the next meeting.
- Major decisions will be documented in GitHub.

### File Naming Convention
- Use lowercase letters.
- Separate words using hyphens (`-`).
- Example:
  - `idea-log.md`
  - `team-profile.md`
  - `prototype-design.png`

### GitHub Workflow
- Create meaningful commits after completing a task.
- Use descriptive commit messages.

Examples:
- `add team profile`
- `update idea log`
- `create homepage wireframe`
- `add project requirements`

### Quality Standards
- Review documentation before submission.
- Ensure all team members understand project progress and deliverables.
- Maintain clear and consistent documentation throughout the semester.

### Conflict Resolution
1. Discuss issues openly during team meetings.
2. Seek a compromise that benefits the project.
3. If agreement cannot be reached, a majority vote will be used.
4. The final decision will be documented for future reference.

## Team Goal

To successfully design and develop a functional IT venture prototype that demonstrates creativity, technical feasibility, and real-world value while strengthening the team's skills in software development, design, collaboration, and project management.
